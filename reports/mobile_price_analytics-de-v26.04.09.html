Bronze Layer

  • Policy: Append-only, zero transformation : every source record stored exactly as received.
  • Rows: 15 records extracted from source.
  • Purpose: Enables full pipeline reprocessing without re-querying the source system : historical backfills and schema change recovery.
  • Access: Engineering team only : not accessible to analytical consumers due to raw data quality.

Silver Layer

  • Rows: 15 records (100.0% retention from Bronze) : 100% source fidelity maintained.
  • Null Handling: No nulls in source : no handling required.
  • Deduplication: Not required : all source rows unique.
  • PII Masking: Not required : no sensitive columns detected.
  • Audit Columns Added: ingestion_timestamp, transformation_timestamp, source_file_name, pipeline_run_id, record_status.

Gold Layer

  • Rows: 13 aggregated groups : grain changed from individual records to dimensional summaries.
  • Aggregation Dimensions: Standard dimensional grouping by available categorical columns.
  • Consumers: BI dashboards, business reporting tools, executive analytics.
  • Refresh Strategy: Full reload on each pipeline run : deterministic, consistent results every execution.
